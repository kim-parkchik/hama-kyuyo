## プロジェクト名について
本プロジェクトは「はま給与（hama-kyuyo）」としてスタートしましたが、現在は **「Q」** という名称で開発を進めています。

- **由来**: 給与の「給（きゅう）ちゃん」から。
- **コンセプト**: 
  - **Quick**: ローカル動作ならではの素早さ
  - **Quality**: 確かな計算精度
  - **Quest**: 理想の給与計算環境の追求
  
（当初は「給ちゃん」という愛称を検討していましたが、親しみやすさを残しつつ、よりシンプルでプロ向けの道具感を出すために一文字の「Q」に決定しました。）

## 開発の背景
市場の給与計算ソフトがクラウド・サブスク型に偏る中、個人事業主や小規模事業者、そして若いアントレプレナーにとって、データの自己管理とランニングコストの削減は重要な課題です。
外部サービスに依存せず、自分のPC一台で完結する、シンプルでコントロール可能な給与計算環境を目指し、本ツールを自作しています。

## 技術スタック
ローカルでの動作を前提に、AIとの対話を通じて選定した構成です。

- **Runtime**: [Bun](https://bun.sh/) (高速な実行環境)
- **Framework**: [Tauri](https://tauri.app/) (デスクトップアプリ化)
- **Frontend**: React + TypeScript (UI構築)
- **Icons**: Lucide React (軽量で一貫性のあるアイコンライブラリ)
- **Database**: SQLite (ローカルデータ保存)

## プロジェクトの特色：AIとの共創
本プロジェクトは、実務家（非プログラマ）が生成AIを「技術アドバイザー」としてフル活用し、設計から実装までを行う**「AI共創型開発」**の実践です。

- **実務知とITの融合**: 給与計算の実務知識を軸に、生成AIの技術支援を受けることで、専門外であったモダンな技術スタックでの開発を実現。
- **文系AI活用の実践**: 既存のソフトを「消費」するだけでなく、生成AIをパートナーに「作る側」へ回ることで、給与計算ロジックの透明性と自由度を自らの手に取り戻す試みです。

## 開発の状況
現在は基本機能（会社設定・従業員管理・勤怠計算の基礎）を実装中です。
「Q」を実戦投入可能な「真の給与計算ソフト」へと進化させるため、以下の高度な実務要件を順次実装・対応予定です。

- **給与計算エンジンの高度化**:
  - **社会保険料の適正管理**: 定時決定（算定基礎）および随時改定（月変）ロジックの実装（標準報酬月額の固定・更新管理）。
  - **所得税の区分対応**: 甲欄・乙欄の判別ロジックの実装。
  - **端数処理・日割り計算**: 月途中入退社時における基本給の日割り、および端数処理の厳密な制御。
- **実務に即した例外対応**:
  - **役員報酬**: 雇用保険の対象外設定（二線引き対応）。
  - **高齢者対応**: 70歳以上被用者（厚生年金免除・健康保険継続等）の自動判定。
  - **休業対応**: 産休・育休中の社会保険料免除期間のステータス管理。
- **住民税の年度・一括管理**: 
  - 市町村別の税額管理に加え、毎年6月の特別徴収税額通知に対応した「一括インポート・年度更新」機能。
- **外部データ連携**:
  - **全銀協フォーマット出力**: ネットバンキングでの一括振込を可能にするFBデータ（全銀形式）生成機能。
- **帳票・法定書類**:
  - 法定三帳簿（労働者名簿・賃金台帳・出勤簿）および源泉徴収票の出力対応。
- **年末調整・マイナンバー**:
  - 年間の所得税精算ロジックの実装。
  - ローカルDBの秘匿性を活かしたマイナンバー管理（データの暗号化・物理分離検討中）。

## 開発の目的
- 「自分が作った」と言える成果物をひとつ完成させること
- クラウドに依存しない、安全で自由な給与計算環境の提供

## 今後の展望：スタンドアロン・アプリへの展開
本プロジェクトは、最終的に **「インストーラーや.exeファイルをダブルクリックするだけで、誰でもすぐに使い始められるデスクトップアプリ」** として配布・運用することを目指しています。

- **環境構築不要**: コマンド操作（Bunのインストール等）を必要とせず、一般的なソフトと同様の直感的な導入を実現します。
- **完全ローカル完結**: ネット環境に依存せず、PCを開けば即座に業務を開始できる「究極の道具感」を追求します。
- **独自ファイル形式の採用**: 会社ごとのdbを .qp（Q Payroll の略）という独自拡張子のファイル単位で管理・保存できる仕組みを目指します。
  - 実務に即した管理: ファイルをダブルクリックするだけで特定の会社データが開く直感的な操作感を実現します。
  - ポータビリティ: データのバックアップや、PC間での受け渡しを「1つのファイル」として容易に行えるようにします。

## 動作環境について
本プロジェクトは、macOS 環境での開発・検証を行っています。
Windows / Linux 環境での動作確認は行っていません。
